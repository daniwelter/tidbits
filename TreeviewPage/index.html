PLATO: Plugin for Autocomplete on Ontologies

Visualising ontologies and locating terms (e.g. within a query interface, or when applying annotations to data) is a common problem that would benefit from a general solution. We developed PLATO, (Plugin for Autocomplete on Ontologies) which is a general-use embeddable JavaScript widget1 used for browsing and searching ontologies. It was purpose-built to be generically configurable for use with any ontology served by the Ontology Lookup Service (OLS). PLATOâ€™s features include the ability to visualise matching ontology terms in their rightful place in the tree, and to expand and collapse nodes as desired. Below, alpha versions of PLATO are deployed for three different ontologies. Note that while the widget can be incorporated within a data query interface, this additional functionality is not yet demonstrated below. We welcome your questions and feedback; please contact jmcmurry [at] ebi.ac.uk.

Experimental Factor Ontology

EFO is an application ontology including ChEBI ontology, Gene Ontology (GO), Uber anatomy ontology (UBERON), Phenotype And Trait Ontology (PATO). It also cross references terms from controlled vocabularies including NCBI taxonomy, Medical subject headings (MeSH), and SNOMED clinical terms.

Cellular Microscopy Phenotype Ontology

CMPO is an ontology of observations where the phenotype is described in terms of a biological process, a cell population, a whole cell or a subcellular structure that bears a particular quality.

EDAM Data and Methods Ontology

EDAM is an ontology of common bioinformatics types of data including identifiers, data formats, operations and topics.
